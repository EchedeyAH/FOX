Feature

  • Device file.
  • Open, close, and ioctl commands.
  • Read/write registers.
  • Signaling IRQ.
  • Dynamic device allocation.
  • Dynamic major number.
  • One major number for multiple devices.
  • Use the GNU configure and build system.

Installation

  • Download and save IXPCI package to a directory which you have access to.
  • Extract the package. For example, the package's file name is ixpci-0.6.0.tar.gz and its path related to your current working directory is ../pkg, then the extraction command would be
    	gunzip -c ../pkg/ixpci-0.6.0.tar.gz | tar xvf -

		or

	tar -zxvf ../pkg/ixpci-0.6.0.tar.gz
  • An ixpci-0.6.0 directory is created after extraction. For convienient access, it is a good idea to put a symbol-link on it.
    	ln -s ixpci-0.6.0 ixpci
  • Change working directory to the ixpci you just made, type ./configure to create proper Makefiles.
  • Once the configuring has done successfully, type make to build all.
  • If you like to install files to system directory, the make install will do it for you. However, install files to system directory is not necessary for further operation. You will need the root privilege for that.
  • Script ./ixpci.inst loads modules automatically. Script ./ixpci.remove removes the loaded modules. The root privilege is required when installing or removing these kernel modules.
  • Please read file README if it is there.
